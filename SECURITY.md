# Security policy

## Supported version

Security fixes are applied to the `main` branch. Historical branches are retained for context and are not supported deployment targets.

## Reporting a vulnerability

Use GitHub's private vulnerability reporting option on the repository's **Security** tab when it is available. If it is not available, contact the repository owner through an existing private channel before opening an issue.

Do not include credentials, personal data, database records, live exploit payloads, or sensitive endpoint details in a public issue. Include the affected revision, impact, reproduction conditions, and a minimal non-destructive demonstration in the private report.

If a credential may have been exposed, revoke or rotate it immediately; removing it from Git history is not sufficient.
